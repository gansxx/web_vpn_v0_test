# VPN连接问题解决指南 🔌
无法连接到VPN服务器？本指南将帮助您快速诊断和解决连接问题。

## 🚨 快速诊断清单
在深入排查前，请先完成以下检查：
- [ ] 确认已登录账户且套餐未过期
- [ ] 确认设备能正常访问互联网
- [ ] 尝试重启VPN客户端
- [ ] 检查是否有杀毒软件阻止连接

## ❌ 连接失败问题

### 症状表现
- 客户端显示"连接失败"或"连接超时"
- 无法获取服务器列表
- 订阅更新失败

### 解决方案

#### 1. 检查防火墙设置
Windows系统经常会阻止VPN客户端的网络访问：

```
操作步骤:
1. 按 Win+R，输入 control panel
2. 选择"系统和安全" → "Windows Defender 防火墙"
3. 点击"允许应用或功能通过Windows Defender防火墙"
4. 找到v2rayN.exe并勾选"专用"和"公用"网络
```

#### 2. 切换网络环境
有时网络环境会影响VPN连接：
- **使用手机热点测试**: 切换到手机网络查看是否能连接
- **更换网络**: 尝试连接其他WiFi网络
- **联系运营商**: 某些网络运营商可能限制VPN流量

#### 3. 更新订阅链接
订阅链接可能已过期或被更新：
1. 重新获取订阅链接
2. 删除旧的配置文件
3. 重新导入新的订阅链接

## 🔄 连接不稳定问题
### 症状表现
- VPN连接时断时续
- 需要频繁手动重连
- 网络时快时慢

### 稳定性优化

#### 1. 客户端设置调整
在v2rayN中进行以下设置：
```
参数 → 路由设置：
- 域名解析策略: IPIfNonMatch
- 启用路由高级功能

参数 → 其他设置：
- 开启TUN模式
- 启用Mux多路复用
```

#### 2. 服务器选择策略
- 选择延迟在100ms以下的服务器
- 避开高峰期的热门节点
- 定期测试不同地区的服务器

#### 3. 网络环境优化
- 使用有线连接替代WiFi
- 关闭其他占用带宽的应用
- 更新网卡驱动程序

## 🔧 高级排查方法
### 网络连通性测试
使用命令行检查网络连通性：
```bash
# 测试基础网络连接
ping 8.8.8.8

# 测试域名解析
nslookup google.com

# 查看路由表
route print
```

### 客户端日志分析
1. 在v2rayN中开启日志记录
2. 查看日志文件中的错误信息
3. 根据错误码查找具体原因

### 端口检测
某些网络环境可能屏蔽特定端口：
```bash
# 检查端口是否开放 (Windows PowerShell)
Test-NetConnection -ComputerName 服务器地址 -Port 443
```

## 📞 仍然无法解决？
### 联系技术支持时请提供：
- 操作系统版本（Windows 10/11等）
- 客户端版本号
- 错误信息的完整截图
- 网络环境描述（家庭/公司/学校网络）
- 已尝试的解决方法

### 联系方式：
- **在线工单**: 控制面板提交详细问题描述
- **技术支持群**: Telegram用户群组
- **实时客服**: 网站右下角客服窗口

> 💡 **温馨提示**: 90%的连接问题都与防火墙设置或网络环境有关。请优先检查这两项设置！
