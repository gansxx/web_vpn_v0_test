## 📁 文档规范

### 命名规范
- 开发日记：`[action]-[module]-YYYY-MM-DD.md`
- 功能文档：`[feature]-guide.md`
- API 文档：`api-[module].md`

### 内容规范
每个开发日记应包含：
1. 任务背景
2. 目标和范围
3. 架构设计
4. 实施步骤
5. 成果对比
6. 质量验证
7. 后续规划
8. 经验总结
