# 故障排除指南 🔧

遇到连接问题？无法访问网站？网速很慢？本指南将帮助您快速解决常见的VPN使用问题。

---

## 🚨 紧急故障处理

### 连接失败 - 立即检查
1. **验证登录状态** - 确认您已正确登录账户
2. **检查套餐状态** - 确认您的VPN套餐未过期
3. **测试网络连接** - 确保您的设备能正常上网
4. **重启客户端** - 完全关闭并重新打开VPN客户端

## 🔍 常见问题诊断

### ❌ 问题：无法连接到VPN服务器

**症状表现:**
- 客户端显示"连接失败"
- 连接超时
- 无法获取服务器列表

**解决方案:**

1. **检查防火墙设置**
   ```
   Windows: 控制面板 → 系统和安全 → Windows Defender 防火墙
   添加 v2rayN.exe 到允许列表
   ```

2. **切换服务器节点**
   - 尝试连接不同地区的服务器
   - 选择延迟较低的节点
   - 避开高峰期使用繁忙节点

3. **检查网络环境**
   - 确认路由器没有屏蔽VPN流量
   - 尝试使用手机热点测试
   - 联系网络运营商确认是否有限制

---

### 🐌 问题：网速很慢

**症状表现:**
- 网页加载缓慢
- 视频卡顿
- 下载速度很慢

**优化方案:**

1. **选择最优服务器**
   - 使用延迟测试功能
   - 选择地理位置较近的服务器
   - 避免选择负载较高的节点

2. **调整路由规则**
   ```
   推荐设置:
   - 国内网站: 直连
   - 国外网站: 代理
   - 广告过滤: 启用
   ```

3. **优化网络设置**
   - 关闭其他占用带宽的应用
   - 使用有线连接替代WiFi
   - 重启路由器刷新网络状态

---

### 🚫 问题：无法访问特定网站

**症状表现:**
- 部分网站无法打开
- 显示"网站无法访问"
- 页面加载不完整

**解决步骤:**

1. **检查路由配置**
   - 确认目标网站走代理路由
   - 更新PAC规则列表
   - 手动添加网站到代理列表

2. **清除DNS缓存**
   ```bash
   Windows: ipconfig /flushdns
   Mac: sudo dscacheutil -flushcache
   ```

3. **更换DNS服务器**
   ```
   推荐DNS:
   - 8.8.8.8 (Google)
   - 1.1.1.1 (Cloudflare)
   - 114.114.114.114 (国内)
   ```

---

### 🔄 问题：连接频繁断开

**症状表现:**
- VPN连接不稳定
- 需要频繁重连
- 网络时断时续

**稳定性优化:**

1. **调整客户端设置**
   - 启用自动重连功能
   - 增加连接超时时间
   - 关闭网络状态检测

2. **网络环境优化**
   - 检查WiFi信号强度
   - 更新网卡驱动程序
   - 关闭节能模式

3. **系统设置调整**
   - 禁用系统代理自动检测
   - 关闭网络位置感知
   - 设置静态IP地址

## 🛠️ 高级故障排除

### 📊 网络诊断工具

1. **延迟测试**
   ```bash
   ping google.com
   ping 8.8.8.8
   ```

2. **路由追踪**
   ```bash
   tracert google.com
   ```

3. **端口检测**
   ```bash
   telnet 服务器IP 端口号
   ```

### 🔧 客户端重置

如果问题持续存在，可以尝试完全重置客户端:

1. **备份配置**
   - 导出订阅链接
   - 记录自定义设置

2. **清理客户端**
   - 删除配置文件夹
   - 清除注册表项（Windows）
   - 重新下载安装

3. **重新配置**
   - 导入订阅链接
   - 恢复个人设置
   - 测试连接状态
